The smart mining roadmap (how the tracker items fit together)

 

Think of it as 7 workstreams that must lock together:

 

) Data capture (OT layer: PLC, scales, load cells)

 

These are the “senses” of the mine.

 

 Determine PLC capabilities (In Progress)

 

Meaning:Confirm what the PLC can expose (signals/tags/events), how often, and how reliably.

 

Process:get access → scan/read tags → agree what matters → sign off the “data list”.

 

 Connect Belt Scales to Server (Done)

 

Meaning:Throughput measurement is flowing to a central point.

 

 Connect Weighbridge load cells to Server (In Progress)

            Meaning:Certified-ish weight measurements can be captured centrally (often needed for reconciliation/controls).

 

Process:agree wiring/IO + protocol → test readings → calibrate → validate with ops.

 

 Data analysis from scales (In Progress)

 

Meaning:The numbers exist — now you check if they’re useful, consistent, and match reality.

 

 Build database from PLC data (In Progress)

 

Meaning:You’re building the “brain’s memory” so dashboards aren’t just live numbers but trends, history, comparisons.

 

Where YOU accelerate:define the KPI dictionaryearly (what each metric means + units + update rate + who trusts it). That prevents weeks of “but what does this number mean?” later.

 

) Infrastructure foundation (servers, services, network, power)

 

This is the boring-but-sacred stuff. No foundation = no dashboard.

 

 Configure POC server (Done)/ Configure server services (Done)

            Meaning:The pilot environment exists (core compute + services).

 

 Manage data traffic to internal network + submit for approval

            (Done)

 

Meaning:IT governance is being respected (good — avoids getting shut down later).

 

 Network layout approved (Done)/ Network infra costing (Done)

            Meaning:The network design is understood + priced.

 

 Hosting solution for all servers/software (In Progress)

            Meaning:Decide where the platform lives (on-prem, cloud, hybrid) and who owns uptime.

 

 SAN drive replacement (In Progress)

 

Meaning:Central storage upgrade (performance + reliability).

 

 Backup hot site (In Progress)

 

Meaning:Disaster recovery — if the main site dies, you can recover fast.

 

 Woest server room power issue (In Progress)

 

Meaning:If power is unstable, everything above becomes a house built on jelly.

 

Where YOU accelerate:make a -page “MVP architecture diagram”(super visual) showing: sources → network → server → DB → dashboard. Executives love pictures. Engineers love clarity. Everyone wins.
